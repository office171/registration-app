# איך להפעיל ולפרסם את מערכת הרישום

האפליקציה בתיקייה הזו היא אתר סטטי: HTML/CSS/JS. אין צורך ב-Node או בבילד כדי לפתוח אותה.

## 1. הקמת Supabase

1. פתח פרויקט חדש ב-Supabase.
2. עבור ל-SQL Editor.
3. הרץ את הקובץ:

   `../supabase-registration-schema.sql`

4. עבור ל-Storage.
5. צור bucket בשם:

   `registration-documents`

6. בשלב ראשון לבדיקה, אפשר להשאיר את הכתיבה דרך anon פתוחה רק אם אתה מבין את הסיכון. בפרודקשן עדיף להעביר שליחה דרך API server-side עם service role.

## 2. הגדרת חיבור

1. העתק את הקובץ:

   `config.example.js`

2. שנה את שמו ל:

   `config.js`

3. מלא בו:

   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `STORAGE_BUCKET`

את הערכים מוצאים ב-Supabase תחת:

Project Settings -> API

## 3. בדיקה מקומית

אפשר לפתוח את `index.html` בדפדפן.

אם הדפדפן חוסם העלאות או קריאות בגלל מקור מקומי, אפשר להפעיל שרת סטטי פשוט. לדוגמה אם יש Python:

```bash
python -m http.server 8000
```

ואז לפתוח:

`http://localhost:8000`

## 4. פרסום כקישור באינטרנט

הכי פשוט:

### Netlify Drop

1. ודא שיש `config.js` אמיתי בתיקייה.
2. גרור את כל תיקיית `registration-app` ל-Netlify Drop.
3. תקבל קישור ציבורי.

### Vercel

1. העלה את התיקייה ל-GitHub.
2. חבר את GitHub ל-Vercel.
3. פרוס כאתר סטטי.

### אחסון רגיל

אפשר להעלות את הקבצים לכל שרת שמגיש קבצים סטטיים:

- `index.html`
- `styles.css`
- `app.js`
- `config.js`

## 5. דברים שחייבים להקשיח לפני שימוש אמיתי

- להגדיר RLS ב-Supabase בצורה מדויקת.
- לא לאפשר לכל גולש לקרוא את טבלת ההרשמות.
- לשקול שליחה דרך API server-side במקום ישירות מהדפדפן.
- להגדיר מגבלות גודל וסוג קובץ ל-Storage.
- להוסיף דשבורד פנימי לצוות.
- להוסיף הודעות מייל אוטומטיות אחרי שליחה/חסר פרטים/אישור.

## 6. מה כבר עובד בקבצים האלה

- טופס רב-שלבי בעברית.
- RTL מלא.
- ולידציה בסיסית.
- שדות מותנים.
- שמירת טיוטה מקומית.
- שליחה לטבלת `registrations`.
- העלאת קבצים ל-Supabase Storage.
- רישום קבצים בטבלת `registration_documents`.

## 7. מה עדיין צריך להוסיף

- דשבורד ניהול.
- Magic link לטיוטה בין מכשירים.
- מיילים אוטומטיים.
- תשלום, אם תרצה.
- עיצוב סופי לפי המיתוג שלכם.
